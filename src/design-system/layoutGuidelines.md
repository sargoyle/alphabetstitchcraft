# Layout Guidelines

## Purpose

Layouts should support clear scanning, responsive behaviour and reliable interaction across desktop and smaller screens.

## Rules

- Use the full available viewport width for app surfaces.
- Keep primary work areas visually dominant.
- Keep tool controls close to the thing they affect.
- Use responsive grids that collapse before text overlaps or buttons wrap awkwardly.
- Allow large previews and table surfaces to scroll horizontally instead of shrinking into unreadable states.
- Keep repeated cards at `8px` radius unless an existing app pattern requires otherwise.

## Recommended Patterns

- Page header: compact title, short supporting copy and one clear primary action.
- Workspace: sidebar controls plus main preview where the task requires two-pane editing.
- Card grid: responsive repeat columns with a minimum usable card width.
- Table/game surface: full-width framed play area with decks, hand, log and status areas clearly separated.
